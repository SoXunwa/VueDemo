var 和 let 的却别

//js引擎在读取js代码时会进行两个步骤，第一个步骤是解释，第二个步骤是执行。
//解析：查找var 和 function，预先声明 var 和 定义函数，因为 会预先定义函数，
//     所以会出现 函数提升的情况（在定义函数前调用不会报错，并且正常执行）

//var 支持在同作用域内重复声明同一变量，会预解析（在执行前，先搜索并声明var），变量提升，换句话说，代码中var声明多了，会降低代码执行速度，
//    如果代码中存在该变量，该变量在初始化前被调用，不会报错，只会输出 undefined，因为预解析在执行前声明了该变量（变量提升），默认值是 undefined

//let 不支持在同作用域内重复声明同一变量，不会预解析，所以不会变量提升，禁止声明前访问该变量，换句话说，会降低代码执行速度，
//    块（代码块）作用域内有效，
//    如果代码中存在该变量，该变量在初始化前被调用，会报错。
